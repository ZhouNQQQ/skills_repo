# 技能审查清单

在最终确定任何技能之前使用此清单。

## 结构

- [ ] 描述包含明确的触发条件（"当……时使用"）
- [ ] 描述在 1024 字符以内
- [ ] SKILL.md 在 100 行以内
- [ ] 溢出内容已移到 references/
- [ ] 引用使用相对路径链接，不是内联

## 内容质量

- [ ] 没有时间敏感信息（日期、版本号、"最近"）
- [ ] 全文术语一致
- [ ] 包含具体示例
- [ ] 每个工作流都有明确的进入和退出条件
- [ ] 多步骤流程使用检查清单
- [ ] 祈使语气（"做 X"，不是"你应该做 X"）

## 反模式检查

- [ ] 不是厨房水槽（一个技能，一个职责）
- [ ] 不是文字墙（渐进式披露）
- [ ] 描述不模糊（具体触发条件）
- [ ] 不是仅面向人类（Agent 优先写作）
- [ ] 不过时（没有时间敏感数据）
- [ ] 没有隐藏依赖（显式的硬/软依赖）
