# AI辅助深度阅读 Skill

将AI视为"第二大脑"，通过三步法把一本书从"看完"变成"吃透"，实现知识的真正内化与深度记忆。

## 核心原则

- **读过≠得到，看完≠掌握**：阅读的有效标准不是"又读完了一本"，而是"有多少内容真正留在了脑子里"
- **AI是第二大脑，不是替代者**：AI不替人读书，而是帮人更快理解、更准提炼、更深记住
- **主动参与是关键**：被动浏览只会产生"假阅读"，只有建立结构、带着问题、主动回忆才能形成深度记忆

## 三步阅读法（核心工作流）

每本书/材料都按以下顺序执行三步。根据用户当前所处阶段，可从任意一步开始。

### 输出前评估（关键决策点）

在生成任何输出前，先评估用户情境，决定输出包的规模和重点：

| 用户情境 | 输出规模 | 重点文件 |
|----------|----------|----------|
| 刚开始读新书（有目录） | 完整包：Step1 全部 + 预创建 Step3 框架 | 阅读指南、阅读计划、概念速查 |
| 刚开始读新书（无目录） | 精简包：搜索目录后执行完整包 | 同上 |
| 正在读某章节 | 章节包：Step2 全部 | 探究问题、场景练习 |
| 刚读完一章/部分 | 内化包：Step3 全部 | 主动回忆、记忆卡片、应用指南 |
| 读完全书想系统复习 | 复习包：Step3 增强版 + 跨章节联想 | 知识总结、综合记忆卡片、应用回顾 |
| 只想快速了解一本书 | 速览包：Step1 精简 + 核心观点摘要 | 阅读指南（精简版） |
| **不知道读什么书（问题驱动）** | **探索包：先问问题 → 推荐书+章节** | **问题诊断、书单推荐、速读启动** |
| **想读但压力大/焦虑** | **微读包：只推荐一小节（10分钟）** | **单节导读、1个探究问题、行动卡片** |

**重要**：默认产出完整包，除非用户明确要求精简。宁可内容多一些让用户选择，也不要遗漏有价值的辅助材料。

### 用户引导与状态导航（MANDATORY）

**这是整个skill最关键的部分。每次与用户交互，必须在回复开头提供清晰的状态导航。**

#### 首次接触引导（当skill第一次被触发时）

第一次与用户交互时，必须先展示以下引导（用自然语言，不要生硬模板化）：

```
我来帮你用三步法把这本书真正吃透。整个流程是这样的：

📍 第一步：建立地图 → 先看清全书骨架，不迷路
📍 第二步：带着问题读 → 变被动为主动，强制大脑参与
📍 第三步：主动回忆 → 读完立刻检验，把知识固化成记忆

你现在在哪？可以告诉我：
• "刚开始读《XX》，给你目录" → 我帮你走第一步
• "正在读第X章" → 我帮你生成探究问题
• "刚读完一章" → 我帮你做主动回忆
• "读完了想复习" → 我帮你系统回顾
```

**规则**：
- 不要假设用户知道三步法——必须先展示全貌
- 不要直接问"你需要什么"——给出明确的选项让用户选
- 如果用户已经提供了明确上下文（如"我刚读完第三章"），则跳过引导，直接进入对应步骤

#### 每步结束后的导航提示（每步完成后必须附加）

每个步骤完成后，回复末尾必须附加导航提示，格式如下：

```
---
✅ 你已完成：Step X — [步骤名称]
📦 我为你生成了：[文件清单，一句话说明每个文件的用途]

👉 接下来你有几个选择：
1. [下一步骤的具体描述] → 告诉我"[触发短语]"
2. [其他可能的选择1] → 告诉我"..."
3. [其他可能的选择2] → 告诉我"..."
```

**具体导航内容**：

**Step 1完成后**：
```
👉 接下来你有几个选择：
1. 进入第X章的深度阅读 → 告诉我"我要读第X章"，并把章节内容发给我
2. 先看阅读计划再开始 → 直接查看上面的 04_阅读计划.md
3. 调整某个部分 → 告诉我具体需要调整什么
```

**Step 2完成后**：
```
👉 接下来你有几个选择：
1. 开始阅读，读完后做主动回忆 → 读完后告诉我"我读完了，帮我做回忆"
2. 这章太难了，需要更多解释 → 告诉我具体哪里不懂
3. 换一章继续 → 告诉我"我要读第X章"
```

**Step 3完成后**：
```
👉 接下来你有几个选择：
1. 继续读下一章 → 告诉我"我要读第X章"
2. 复习之前的内容 → 告诉我"帮我复习"，我会出跨章节联想问题
3. 这本书读完了，想系统回顾 → 告诉我"读完了，帮我总结"
4. 这章没帮助/想换方向 → 告诉我"这章不太对"，我会帮你调整推荐
```

#### 当前状态行（每次回复都显示）

除了首次引导和步骤结束导航，每次回复开头应包含一行状态标识：

```
📖 《书名》| 当前阶段：Step X（共3步）| 已完成：第X章 / 共XX章
```

如果用户同时在读多本书，显示当前活跃的书名。

### Step 1: 先看结构，建立阅读地图

**目的**：在深入正文前，先看清全书骨架，做到"先有地图，再上路"。

**触发条件**：
- 用户刚开始读一本新书
- 用户提供了书籍目录
- 用户需要理解一本书的整体框架

**执行步骤**：

1. 评估用户情境（见"输出前评估"），确定输出包规模
2. 获取书籍的完整目录/大纲（由用户提供，或通过工具搜索）
3. 将目录发给AI，使用提示词模板：`references/prompts.md` 中的 "结构分析提示词"
4. AI分析后，产出 **Step 1 输出包** 中的全部文件：
   - `01_阅读指南.md`：全书骨架 + 阅读策略 + 预期收获
   - `02_章节导读.md`：每章预读问题（3-5个/章）
   - `03_核心概念速查.md`：创建框架，预留15个核心概念条目
   - `04_阅读计划.md`：分阶段计划（含3周/6周/自定义版本）
5. 向用户呈现完整的阅读 companion 套装

**质量标准**：
- 必须提炼出全书的顶层框架（不超过7个核心模块）
- 必须说明各部分之间的逻辑关系
- 必须标出建议优先阅读的重点章节
- 阅读计划必须包含具体周数和每日阅读量的建议
- 章节导读必须为每个重点章节提供预读问题
- 概念速查表必须预留至少15个条目（即使尚未填充内容）

---

### Step 2: 带着问题读，变被动为主动

**目的**：将阅读从"浏览文字"转变为"寻找答案"，强制大脑主动参与。

**触发条件**：
- Step 1完成后，准备进入具体章节阅读
- 用户需要深入理解某个章节
- 用户希望检验自己对内容的理解深度

**执行步骤**：

1. 评估用户情境（见"输出前评估"），确定输出包规模
2. 根据Step 1的结构分析，确定当前要阅读的章节
3. 将章节内容/主题发给AI，使用提示词模板：`references/prompts.md` 中的 "问题生成提示词"
4. 产出 **Step 2 输出包** 中的全部文件：
   - `01_本章分析.md`：章节结构 + 核心论点 + 论证逻辑
   - `02_探究问题.md`：8-12个深度思考问题（开放式，非简单事实问答）
   - `03_场景练习.md`：2-3个实践场景 + 应用练习题
   - `04_概念笔记.md`：本章新概念的详细笔记（填充到概念速查表）
5. 用户带着探究问题去阅读（或让AI朗读/总结后回答）
6. 用户回答后，AI给予反馈和补充

**问题设计原则**：
- 避免简单事实性问题（如"作者叫什么名字"）
- 聚焦"为什么"和"如何应用"（如"为什么说痛苦+反思=进步？"）
- 关联读者自身经验（如"你在工作中遇到过类似的决策困境吗？"）
- 每个问题应能引发深度思考，而非简单记忆
- 探究问题数量：8-12个，确保覆盖本章所有核心论点

---

### Step 3: 读完后做"主动回忆"，固化记忆

**目的**：通过"提取练习"将短期理解转化为长期记忆。这是记忆真正形成的时刻。

**触发条件**：
- 读完一章或一个完整部分后
- 用户希望检验记忆效果
- 用户需要复习已读内容

**执行步骤**：

1. 评估用户情境（见"输出前评估"），确定输出包规模
2. 将已读章节的核心内容/笔记发给AI
3. 使用提示词模板：`references/prompts.md` 中的 "主动回忆提示词"
4. 产出 **Step 3 输出包** 中的全部文件：
   - `01_知识总结.md`：结构化总结本章核心内容
   - `02_主动回忆.md`：8-10个回忆性问题 + 回答模板 + 反馈框架
   - `03_记忆卡片.md`：10-15张记忆卡片（含概念、含义、应用场景、记忆钩子）
   - `04_应用指南.md`：3-5个实践场景 x 行动步骤 + 30-60-90天计划
   - `05_自我反思.md`：反思模板 + 知识关联 + 个人行动清单
   - **`06_行动卡片.md`：聚焦到"改一个动作"的精简输出（见下方行动卡片规范）**
5. **关键动作**：在主动回忆文件中，要求用户先尝试自己回答，不要直接看答案
6. 用户回答后，AI对照原文逻辑进行反馈：
   - 回答准确的 → 强化关联
   - 回答不完整/错误的 → 指出遗漏，重新解释核心逻辑
7. **苏格拉底式提问（可选，见下方规范）**：如果用户回答较浅或未主动分享感悟，用苏格拉底式追问替代标准化的回忆测试

**关于应用指南的详细要求**：
- 每个场景必须包含：情境描述 + 角色设定 + 具体任务 + 成功标准
- 30-60-90天计划必须具体到每周的行动项
- 应用指南必须与书中的具体概念/方法直接挂钩，不能泛泛而谈

**关于行动卡片的规范（新增）**：

行动卡片是 carl-weread 风格的一页式精简输出，聚焦"改一个动作"而非系统规划。格式如下：

```markdown
━━━━━━━━━━━━━━━━━━━━
📇 阅读行动卡 | 《书名》第X章
━━━━━━━━━━━━━━━━━━━━
🎯 一句话收获：[用一句话概括本章最核心的洞察]

❓ 解释了你什么问题：[回应用户最初带的问题——这一节如何回答了你的疑惑]

⚡ 马上改的一个动作：
   [今天/本周可以做的最小、最具体的行动，不超过3行]

📝 可沉淀的笔记：
   [一句话版本，方便后续快速回顾]
━━━━━━━━━━━━━━━━━━━━
```

**行动卡片 vs 应用指南的区别**：
- 应用指南：`04_应用指南.md` → 系统性的多场景深度规划（3-5个场景 + 30-60-90天计划）
- 行动卡片：`06_行动卡片.md` → 一页纸的极简输出，只聚焦**一个**当下可执行的动作
- **两者并存**：用户既需要长期规划，也需要立即行动的抓手

**关于苏格拉底式提问的规范（新增）**：

苏格拉底式提问不是替代主动回忆测试，而是作为**补充对话模式**，适用于：
- 用户回答较浅、只是简单复述
- 用户说"看完了"但没有分享感悟
- 用户看起来没有真正理解核心概念

**苏格拉底提问流程（3-4轮递进）**：

```
第1轮（澄清）："你觉得这一节对你一开始的问题有帮助吗？具体是哪一点？"
第2轮（关联）："如果现在让你用一个新学到的道理给同事/朋友提建议，你会说什么？"
第3轮（深挖）："哪句话让你停下来想了想？为什么是它而不是其他内容？"
第4轮（行动）："基于刚才的讨论，你觉得今天可以做的一件最小的事是什么？"
```

**规则**：
- 苏格拉底式提问和主动回忆测试**二选一执行**，不要同时做两种
- 如果用户主动回答了回忆测试且质量不错 → 用主动回忆模式
- 如果用户回答较敷衍或没有主动分享 → 切换到苏格拉底式追问
- 苏格拉底提问的最终产出也是行动卡片（从对话中提炼）

**为什么这一步不可替代**：
- 划线和笔记≠掌握：能否从脑子里提取出来才是关键
- 主动提取能强化神经通路，形成长期记忆
- "好像看懂了"到"真的能说出来"是质的飞跃

## 输出规范：结构化文件包

每个步骤不应只输出单一文件，而应产出**一个多文件输出包**（3-5个文件），覆盖学习的多个维度。文件包的设计原则是：每个文件有明确的独立用途，用户可以根据当前需要单独查阅。

### Step 1 输出包：阅读 companion 套装

当用户刚开始读一本书时，产出以下文件：

| 文件 | 内容 | 用途 |
|------|------|------|
| `01_阅读指南.md` | 全书骨架分析 + 阅读策略建议 + 预期收获 | 读前建立全局认知 |
| `02_章节导读.md` | 每章/模块的预读问题 + 核心预期 | 读每章前快速预览 |
| `03_核心概念速查.md` | 书中核心概念列表（预留，后续填充） | 读中查阅，读完复习 |
| `04_阅读计划.md` | 分阶段阅读计划（含具体日程和里程碑） | 跟踪进度，确保执行 |

**质量标准**：
- 阅读指南必须提炼全书顶层框架（不超过7个核心模块）
- 阅读计划必须具体到章节级别，有建议完成时间
- 章节导读必须为每个重点章节生成3-5个预读问题
- 概念速查表必须预留足够条目（至少覆盖预期的10-15个核心概念）

### Step 2 输出包：章节深度 companion

当用户深入阅读具体章节时，产出以下文件：

| 文件 | 内容 | 用途 |
|------|------|------|
| `01_本章分析.md` | 章节结构 + 核心论点 + 论证逻辑 | 理解本章说了什么 |
| `02_探究问题.md` | 8-12个深度思考问题（含个人关联问题） | 带着问题读，强制主动参与 |
| `03_场景练习.md` | 2-3个实践场景 + 应用练习题 | 读中暂停练习，促进理解 |
| `04_概念笔记.md` | 本章新概念的详细笔记（加入概念速查表） | 积累到全书知识库 |

**质量标准**：
- 探究问题必须涵盖"为什么""如何应用""与你何干"三类
- 场景练习必须有具体的角色、情境、任务
- 概念笔记必须用用户能理解的语言重新表述

### Step 3 输出包：内化与应用套装

当用户完成阅读后，产出以下文件：

| 文件 | 内容 | 用途 |
|------|------|------|
| `01_知识总结.md` | 结构化知识总结 + 核心论点重述 | 整体回顾，检验理解 |
| `02_主动回忆.md` | 8-10个回忆问题 + 回答模板 + 反馈框架 | 提取练习，固化记忆 |
| `03_记忆卡片.md` | 10-15张记忆卡片（正面概念/背面应用） | spaced repetition 复习 |
| `04_应用指南.md` | 实践场景 x 行动步骤 x 30-60-90天计划 | 从"知道"到"做到" |
| `05_自我反思.md` | 反思模板 + 关联已有知识 + 个人行动计划 | 深度内化，形成洞察 |
| `06_行动卡片.md` | 一页式极简输出：一句话收获 + 一个问题 + 一个动作 | 读完立刻知道该做什么 |

**质量标准**：
- 主动回忆必须强制"先自己回答再看反馈"的流程
- 记忆卡片必须包含：概念、含义、应用场景、记忆钩子
- 应用指南必须包含至少3个具体场景 + 每个场景的行动步骤
- 自我反思必须引导用户将书中内容与个人经验关联
- **行动卡片必须聚焦到"改一个动作"，不能泛泛而谈**
- **苏格拉底式提问或主动回忆测试至少执行其一**

### 跨步骤累积文件

以下文件在多个步骤中持续累积更新：

- **`核心概念速查表.md`**：Step 1 创建框架 → Step 2 填充内容 → Step 3 形成完整速查
- **`阅读日志.md`**：每章的完成记录 + 个人感想 + 待复习标记
- **`行动清单.md`**：从书中提取的具体行动项 + 完成状态 + 截止日期

## 处理变体场景

### 用户已有读书笔记/划线
→ 直接进入Step 3 完整输出包：用已有笔记生成知识总结 + 主动回忆 + 记忆卡片 + 应用指南 + 自我反思 + 行动卡片

### 用户只想快速了解一本书
→ 执行Step 1 精简包：阅读指南（浓缩版）+ 核心观点速览。告知用户如需完整companion可重新触发完整流程

### 用户在阅读过程中卡住
→ 回到Step 2 章节包：针对具体困难段落生成探究问题 + 补充场景练习

### 用户读完多章后想系统复习
→ 执行Step 3 复习包：跨章节知识总结 + 综合记忆卡片（15-20张）+ 跨章节联想问题 + 应用回顾

### 用户反馈"内容太多/太少"
→ 动态调整输出包规模：默认完整包，但可根据用户偏好切换为精简包或扩展包

### 用户想要持续跟踪多本书
→ 为每本书创建独立的阅读文件夹，包含累积文件（概念速查表、阅读日志、行动清单），支持跨书的主题关联

### **用户说"不知道读什么"/没有明确目标书（新增）**
→ **执行探索包**：
1. 先问用户"你现在卡在什么问题里？"或"最近有什么困扰？"
2. 根据用户回答判断：是缺信息、缺框架、还是缺行动方法？
3. 结合用户的阅读历史和兴趣领域，推荐1-2本最匹配的书 + 具体章节
4. 以微读包启动（只推荐一小节），降低启动压力
5. 用苏格拉底式提问引导用户明确真实需求

### **用户读完反馈"这章没帮助"/"看不懂"（新增，反馈闭环）**
→ **启动反馈调整**：
1. 用苏格拉底式提问了解："你觉得没帮助是因为内容太浅、太深、还是方向不对？"
2. 如果方向不对 → 回到探索包，重新诊断问题，推荐不同章节或不同书籍
3. 如果太深 → 推荐同一主题的更基础章节/书籍
4. 如果太浅 → 推荐同一主题的进阶章节
5. **记录用户的反馈**，避免下次推荐同类内容
6. 调整后重新进入Step 2或Step 3

## 质量检查清单

每次完成三步法后，确认：

- [ ] **结构掌握**：用户能不看书记住全书的顶层结构（7个以内核心模块）
- [ ] **概念理解**：用户能用自己的话解释至少5个核心概念
- [ ] **知识提取**：主动回忆覆盖了本章所有核心概念，问题数量在8-10个
- [ ] **卡片系统**：生成了至少10张记忆卡片，每张包含概念+含义+应用+记忆钩子
- [ ] **应用转化**：应用指南包含至少3个具体场景，每个场景有明确的行动步骤
- [ ] **持续计划**：生成了30天以上的复习/应用计划
- [ ] **文件完整性**：产出了该步骤输出包中规定的全部文件类型
- [ ] **跨章积累**：核心概念速查表、阅读日志、行动清单等累积文件已更新
- [ ] **行动卡片**：生成了06_行动卡片.md，聚焦到"改一个动作"
- [ ] **反思深度**：执行了主动回忆测试或苏格拉底式提问（至少其一），用户有真正的思考输出

## 注意事项

- **不要替用户读书**：AI的角色是引导思考，不是提供现成答案
- **强制用户先回答**：Step 3必须要求用户先尝试自己回答，再给出反馈
- **追求全面而非精简**：默认产出完整输出包，用户可以选择性使用；宁可多一些内容，也不要遗漏有价值的辅助材料
- **问题数量的弹性范围**：
  - Step 2 探究问题：8-12个（目标：全面覆盖章节核心论点）
  - Step 3 回忆问题：8-10个（目标：覆盖所有关键概念）
  - Step 3 记忆卡片：10-15张（目标：每张卡片对应一个重要概念或方法）
  - Step 3 实践场景：3-5个（目标：覆盖不同的应用场景）
- **场景化优先**：每个核心概念必须配上至少1个具体的生活/职场场景。不要只解释概念"是什么"，要说明"什么时候用、怎么用"
- **记忆口诀设计**：每张记忆卡片必须包含一个朗朗上口的记忆口诀或类比（如"竹子前四年只长3cm，第五年每天长30cm"），帮助用户长期记住
- **关注应用转化**：始终把"如何用到实际生活中"作为最终目标。应用指南必须是具体、可操作的，不能泛泛而谈
- **多文件优先**：将不同用途的内容拆分到不同文件，方便用户按需查阅
- **迭代优化**：根据用户的反馈和记忆效果，调整问题的深度和角度
- **反馈闭环（新增）**：如果用户反馈"读完没帮助"，必须启动反馈调整流程（见变体场景"用户读完反馈没帮助"），记录原因并调整后续推荐
- **少读一点，读准一点（新增）**：如果用户表现出阅读压力或焦虑，主动降级到微读包（只推荐一小节），用"喝杯咖啡的时间"完成阅读
- **苏格拉底式提问的时机（新增）**：不要对所有用户都使用标准化回忆测试。判断用户回答质量，回答敷衍时切换到苏格拉底式追问
