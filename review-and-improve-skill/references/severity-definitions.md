# 问题严重程度定义

## P1 — 必须修复（阻断性问题）

阻止 skill 正常运作的问题：

| 问题 | 影响 | 修复方法 |
|------|------|----------|
| SKILL.md > 100 行 | 上下文膨胀，触发后占用过多 token | 将溢出内容移至 references/ 或 EXAMPLES.md |
| 缺少 YAML frontmatter | 代理无法解析技能元数据 | 添加标准 `---` 包围的 name 和 description 字段 |
| 无触发条件 | 代理从不加载技能 | 在 description 中添加 "Use when..." 子句 |
| 描述太模糊 | 无法与其他技能区分 | 添加具体细节和触发场景 |

## P2 — 应该修复（质量问题）

降低 skill 有效性的问题：

| 问题 | 影响 | 修复方法 |
|------|------|----------|
| description 触发场景不足 | < 3 个具体场景 | 补充 "创建后 review"、"性能诊断" 等场景 |
| REFERENCE.md > 200 行 | 单文件臃肿，难以按需加载 | 按主题拆分为独立 reference 文件 |
| 内联最佳实践 | SKILL.md 中嵌入应放入 references/ 的详细内容 | 迁移到 references/ 目录 |
| 时间敏感内容 | 包含日期、版本号、即将过时的参考信息 | 用条件语言替换时间引用 |
| 缺少反馈闭环 | 执行修复后无验证步骤 | 增加"重新分析验证"步骤 |
| AIGC 头部污染 | 存在 ReservedCode、ProduceID 等非标准头部 | 清理为标准 YAML frontmatter |
| 重复内容 | SKILL.md 与 references/ 内容重复 | 去重，信息只存一份 |

## P3 — 建议优化（体验提升）

小的改进，不阻塞功能：

| 问题 | 影响 | 修复方法 |
|------|------|----------|
| 脚本位置不规范 | 脚本未放在 skill 级 `scripts/` 目录下 | 迁移到 skill 内部 `scripts/` |
| 缺少 EXAMPLES.md | 复杂 skill 无输入/输出示例 | 添加示例文件 |
| 术语不一致 | 随着时间推移会产生混淆 | 标准化术语 |
