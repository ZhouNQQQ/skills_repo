# personal-learning-coach

## 首次使用引导（6项输入）

Skill 启动后**逐项引导**用户完成初始化，不一次性轰炸：

| 步骤 | 输入项 | 引导话术 | 说明 |
|------|--------|---------|------|
| 1 | **学习资料** | "先传你的学习资料吧~ 可以是附件、链接，或者ima里已有的知识库" | 支持多文件/多链接/ima知识库名称 |
| 2 | **学习计划** | "学习计划是啥？有路线图吗？没有的话我可以帮你从资料里自动生成一个" | 支持附件/链接/ima知识库/自动生成 |
| 3 | **督促形式** | "想让我怎么催你？消息推送/学习日报/考试触发/混合模式？" | 单选 |
| 4 | **督促频率** | "多久催一次？每天/每周/模块完成/进度落后时/自定义时间点？" | 可组合选择 |
| 5 | **笔记格式** | "笔记习惯用什么格式？Markdown/思维导图大纲/纯文本/结构化JSON？" | 决定输出格式 |
| 6 | **笔记存放位置** | "笔记存哪？本地workspace里，还是同步到ima知识库？" | 关键分支 |

## 笔记存放分支逻辑

```
用户选择"本地"
    → 直接生成到 ~/.openclaw/workspace/notes/{学习计划名称}/

用户选择"ima知识库"
    → Step 1: 检查 ima-skill 是否安装
    → Step 2: 验证 API Key 配置
              运行: test -f ~/.config/ima/client_id && test -f ~/.config/ima/api_key
              如果未配置 → 引导用户去 https://ima.qq.com/agent-interface 获取
    → Step 3: 询问用户"知识库名称"（如"Agent学习笔记"）
    → Step 4: 调用 ima-skill 的 knowledge-base 模块写入
```

## 能力模块（13项）

| 模块 | 功能 | 细节 |
|------|------|------|
| **📚 资料库管理** | 接收上传资料，提取目录结构、建立章节索引 | 支持 `.docx`/`.md`/`.pdf`/URL/ima知识库 |
| **🗺️ 计划解析** | 拆解为"阶段→模块→任务→里程碑"四级结构 | 无计划时可基于资料自动生成建议 |
| **📊 进度追踪** | 任务状态记录，理论vs实际进度对比 | 落后项标红，超前项标绿 |
| **⏰ 智能督促** | 按设定频率推送学习任务 | 考虑用户每日可用时长 |
| **🎓 内容速讲** | 精简讲解核心概念 | **必须附带原始资料路径** |
| **📖 原始知识** | 提供资料中的原文段落/完整章节 | 供用户对照学习 |
| **🔍 详细版** | 深入技术细节、代码示例、架构图解释 | 适合深度学习时 |
| **📝 考试模式** | **模块完成后自动出题** | 模仿老师：选择题+简答题+实践题 |
| **📒 笔记管理** | 接收笔记文件，关联章节，全文检索 | 按选择位置存储（本地/ima） |
| **🔎 跨资料检索** | 在全部资料库中搜索知识点 | 聚合多来源相关内容 |
| **🏁 里程碑检查** | 检查阶段核心产出物是否完成 | 未完成则提醒 |
| **🔄 动态调参** | 进度落后/超前时重新排期 | 用户确认后执行 |
| **📈 学习报告** | 生成阶段性报告 | 已完成/待完成/薄弱点/时间投入 |

## 每日督促输出格式

当用户问"今天学什么"或定时触发时，输出结构：

```
📋 今日任务：[模块名] - [任务名]
⏱️ 预计耗时：X小时

━━━ 速讲版 ━━━
[精简核心概念，2-3句话]

━━━ 详细版 ━━━
[技术细节、代码示例、架构解释]

━━━ 原始知识 ━━━
📁 资料位置：[完整文件路径或ima知识库链接]
📄 原文章节：[具体章节名/页码]
```

## 考试模式设计

**触发条件**：用户标记"某模块完成"时自动触发

**题目结构**：
- **概念题**（30%）：资料中定义/原理的理解
- **应用题**（50%）：场景分析、代码补全、架构设计
- **拓展题**（20%）：结合多份资料的综合分析

**输出**：
```
📝 [模块名] 模块测试（共X题，建议用时Y分钟）

一、概念题
1. ...

二、应用题
...

三、拓展题
...

💡 答案将在你回答后给出，也可以直接输入"交卷"
```
