# Skill: 学习计划设计综合入口 (Learning Design Suite)

## 1. 概述

本 Skill 是一个**学习计划编排总入口**，负责串联三个子 Skill，为任何学习场景生成完整、可执行的个性化学习计划。

**触发条件**：当用户提出以下任一意图时激活本 Skill：
- "帮我设计学习计划" / "我想学习XX" / "帮我制定学习方案"
- "如何高效学习某个知识/技能"
- "如何备考XX考试" / "如何在X天内学会XX"
- "优化我的学习方法" / "帮我规划学习路径"
- 任何包含"学习计划、学习方案、备考计划、技能提升、知识学习"的明确指令

### 1.1 三大方法论定位
本 Skill 整合了以下三本书的核心方法论，形成完整学习系统：

| 子 Skill | 核心思想 | 功能定位 | 文件路径 |
|---------|---------|---------|---------|
| **费曼学习法** | 以教代学，输出倒逼输入 | **深度理解**与掌握检验 | `/mnt/agents/output/feynman-learning/SKILL.md` |
| **科学学习底层逻辑** | 基于认知科学的效率优化 | **知识留存**与学习效率 | `/mnt/agents/output/science-learning-core/SKILL.md` |
| **学习力与习惯系统** | 动力系统与习惯养成 | **长期执行**与阻力克服 | `/mnt/agents/output/learning-power-habits/SKILL.md` |

> **编排原则**：本 Skill 不做方法论的重复叙述，只做**场景分析→策略匹配→调用子 Skill→整合输出**。三个子 Skill 分别负责各自领域的具体方法执行。

## 2. 七步学习计划设计工作流

### Step 1: 需求分析 (Demand Analysis)

**目标**：通过结构化提问，明确用户的学习画像。

**必须收集的 5 个维度**：
1. **学习目标 (Goal)**：用户想学什么？期望达到什么水平？（量化描述，如"通过雅思6.5分""独立开发Python爬虫""理解微积分核心概念"）
2. **时间约束 (Time)**：可用总时长？截止日期？每日可投入时间？
3. **当前水平 (Baseline)**：零基础 / 有基础 / 复习提升？相关前置知识掌握情况？
4. **应用场景 (Context)**：应试、工作实践、学术研究、兴趣爱好？
5. **可用资源 (Resource)**：教材、课程、导师、学习伙伴、工具等

**输出格式**：生成一份 `学习需求画像卡片`，包含上述 5 个维度的明确结论。

---

### Step 2: 内容拆解 (Content Decomposition)

**目标**：将宏观学习目标拆分为可执行的知识/技能模块。

**拆解标准**：
- 每个模块应能在 **1 次学习 session（25-50分钟）** 内完成初步学习
- 模块之间应有**明确的依赖关系**（前置→后续）
- 按**难度梯度**排列（基础→进阶→应用）

**输出格式**：
```
知识/技能地图 (Knowledge Map):
├── 模块A（基础层）：xxx
│   ├── 子模块A1
│   ├── 子模块A2
├── 模块B（进阶层）：xxx [依赖模块A]
│   ├── ...
└── 模块C（应用层）：xxx [依赖模块A,B]
```

---

### Step 3: 策略选择 (Strategy Selection)

**目标**：根据内容特征匹配最合适的学习策略组合。

**策略决策矩阵**（由本 Skill 自主判断，并决定加载哪些子 Skill）：

| 内容类型 | 首选策略 | 辅助策略 | 需加载的子 Skill |
|---------|---------|---------|----------------|
| **概念理解型**（理论、定义、原理） | 费曼输出法 | 间隔重复 + 主动回忆 | 费曼学习法 + 科学学习 |
| **技能操作型**（编程、乐器、运动） | 刻意练习 | 示范→模仿→反馈循环 | 科学学习 |
| **记忆密集型**（词汇、公式、条文） | 间隔重复 + 主动回忆 | 记忆宫殿/联想 | 科学学习 |
| **问题解决型**（数学、工程、研发） | PBL项目驱动 | 费曼法梳理思路 | 费曼学习法 + 科学学习 |
| **长期习惯型**（持续学习、终身学习） | 预习-学习-复习闭环 | 动力管理 + 情绪调节 | 学习力习惯 |

**加载规则**：
- 凡涉及"理解深度检验"，加载 **费曼学习法 Skill**
- 凡涉及"效率优化、记忆强化"，加载 **科学学习核心 Skill**
- 凡涉及"长期执行、动力维持"，加载 **学习力习惯 Skill**
- 一个学习计划可能同时加载 1-3 个子 Skill

---

### Step 4: 计划生成 (Plan Generation)

**目标**：输出结构化的、包含时间/任务/方法/评估标准的可执行学习计划。

**计划必须包含 5 个时间维度**：

| 时间维度 | 内容 | 方法要求 |
|---------|------|---------|
| **每日 (Daily)** | 当日学习任务清单 | 明确 session 数量、每个 session 的输入/输出动作 |
| **每周 (Weekly)** | 周目标 + 周回顾 | 包含一次主动回忆测试 + 一次费曼输出 |
| **阶段 (Phase)** | 模块完成节点 | 设置里程碑检查点 |
| **半程 (Mid-term)** | 中期评估与调整 | 对照目标校准进度 |
| **全程 (Full-term)** | 总目标达成 + 复盘 | 完整知识图谱输出 |

**每个学习 Session 的标准结构**（直接嵌入计划）：
```
Session [XX] | [日期] | [时间段]
├── 输入阶段（20-30min）
│   ├── 预习：浏览提纲，标记疑问（参考学习力预习法）
│   └── 学习：主动阅读/听课/练习
├── 输出阶段（10-15min）
│   ├── 费曼输出：用自己的话复述核心概念（加载费曼 Skill 执行）
│   └── 主动回忆：闭眼复述，标记卡壳点（加载科学学习 Skill 执行）
└── 巩固阶段（5min）
    └── 记录到间隔重复系统（Anki/RemNote）
```

---

### Step 5: 资源整合 (Resource Integration)

**目标**：为计划匹配学习材料、工具和环境配置。

**整合清单模板**（根据用户资源情况选择性填充）：

```
📚 学习材料
├── 核心教材：[书名/课程名] — 用途说明
├── 辅助参考：[补充材料] — 使用时机
└── 输出工具：[思维导图/笔记软件/博客] — 费曼输出的载体

🛠 工具配置
├── 间隔重复：Anki / RemNote / Notion（加载科学学习 Skill 的间隔算法）
├── 时间管理：番茄钟 / 时间块 / 日历
├── 进度追踪：看板 / 清单 / 习惯追踪器
└── 环境设置：固定学习地点、设备准备、干扰屏蔽

👥 支持系统
├── 学习伙伴：同伴讨论/互相复述（费曼输出的听众）
├── 导师/社区：答疑渠道、反馈来源
└──  accountability：进度汇报对象
```

---

### Step 6: 执行支持 (Execution Support)

**目标**：为计划执行阶段提供动力和习惯维持机制。

**本阶段必须加载 `学习力习惯 Skill`**，从中提取以下要素嵌入计划：

| 执行挑战 | 引用子 Skill 的解决方案 |
|---------|----------------------|
| 学习动力波动 | 引用：动力维持策略、目标拆解激励 |
| 时间管理失控 | 引用：预习-听课-复习闭环、时间块管理 |
| 情绪焦虑/倦怠 | 引用：情绪管理、压力调节、运动睡眠优化 |
| 拖延/逃避 | 引用：微习惯启动、5分钟法则、环境设计 |
| 社交干扰 | 引用：人际关系边界、学习共同体建设 |

**执行支持模块的输出格式**：
- `动机维护清单`：每周一次的动力检查动作
- `阻力预案`：针对用户最可能的3个放弃场景，提前设计应对策略
- `习惯追踪表`：每日打卡项（可配合任何习惯追踪工具）

---

### Step 7: 评估迭代 (Evaluation & Iteration)

**目标**：建立周期性反馈机制，确保计划持续优化。

**评估指标体系**（从三个子 Skill 中提取评估标准）：

| 评估维度 | 评估方式 | 引用来源 |
|---------|---------|---------|
| 理解深度 | 费曼复述成功率、教学他人反馈 | 费曼学习法 Skill |
| 知识留存 | 间隔重复卡片正确率、主动回忆流畅度 | 科学学习核心 Skill |
| 执行稳定性 | 计划完成率、习惯追踪连续天数 | 学习力习惯 Skill |
| 整体效能 | 目标达成进度、学习投入产出比 | 综合判断 |

**迭代规则**：
- **每周微迭代**：根据完成率和回忆测试结果，调整下周任务量
- **阶段中迭代**：完成一个模块后，评估是否需要增减内容/调整策略
- **半程大迭代**：中期评估时，重新运行本 Skill 的 Step 1-4，生成后半程计划

## 3. 四种典型场景的计划模板

### 3.1 考试备考型（短期高密度）
**特征**：目标明确、时间有限、结果可量化
**策略组合**：
- 科学学习：间隔重复 + 主动回忆（高频）+ 睡眠优化
- 费曼法：用费曼复述梳理知识体系，检验薄弱点
- 学习力：严格时间管理 + 压力调节
**计划结构**：以"倒计时"为主线，每日 3-5 个学习 session，每周 1 次模考/回顾

### 3.2 技能习得型（中期实践导向）
**特征**：需要大量动手、反馈循环、渐进难度
**策略组合**：
- 科学学习：刻意练习（分解子技能、即时反馈）+ 建立连接
- 费曼法：输出项目/作品（教别人如何做）
- 学习力：预习-实践-复盘闭环 + 习惯固化
**计划结构**：以"项目里程碑"为主线，每个阶段产出可展示的作品/成果

### 3.3 知识拓展型（长期广度优先）
**特征**：无硬性 deadline、兴趣驱动、持续积累
**策略组合**：
- 费曼法：定期输出读书笔记/分享文章
- 科学学习：间隔重复维持长期记忆
- 学习力：微习惯设计（每天30分钟）+ 自主学习系统
**计划结构**：以"月度主题"为主线，每月聚焦一个子领域，年底形成知识图谱

### 3.4 问题解决型（目标导向的 PBL）
**特征**：以真实问题为中心、跨学科、探索性强
**策略组合**：
- 费曼法：问题拆解后的逐步澄清与假设验证
- 科学学习：建立连接（跨领域知识嫁接）+ 检索练习
- 学习力：项目管理式学习 + 情绪韧性
**计划结构**：以"问题解决进度"为主线，每个迭代周期定义子问题→学习→验证

## 4. 子 Skill 加载指令

### 4.1 加载时机

在七步工作流的以下节点，按需加载对应子 Skill：

```
Step 1 需求分析 ──────→ 无需加载子 Skill（本 Skill 自主完成）
   ↓
Step 2 内容拆解 ──────→ 可加载费曼 Skill（辅助知识地图结构化）
   ↓
Step 3 策略选择 ──────→ 决策加载哪些子 Skill
   ↓
Step 4 计划生成 ──────→ 必加载对应子 Skill（提取具体方法嵌入计划）
   ↓
Step 5 资源整合 ──────→ 加载科学学习 Skill（工具与间隔算法配置）
   ↓
Step 6 执行支持 ──────→ 加载学习力习惯 Skill（动力与习惯方案）
   ↓
Step 7 评估迭代 ──────→ 按需加载（根据评估维度加载对应 Skill）
```

通过文件路径引用加载：


## 5. 完整执行示例

**用户输入**："我想在2个月内学会Python数据分析，每天2小时，零基础，主要是为了工作中的数据处理"

**本 Skill 执行流程**：

1. **需求分析**：目标=Python数据分析能力；时间=60天/每天2小时；水平=零基础；场景=工作实践；资源=待确认
2. **内容拆解**：Python基础 → NumPy/Pandas核心 → 数据清洗 → 可视化 → 实战项目
3. **策略选择**：技能习得型 → 加载科学学习（刻意练习）+ 费曼法（输出博客）+ 学习力（习惯闭环）
4. **计划生成**：输出60天日程，每个session含"输入+输出+巩固"三段式
5. **资源整合**：推荐教材、Jupyter环境、Kaggle数据集、Anki卡片
6. **执行支持**：加载学习力 Skill → 嵌入动力维持、拖延应对、每日打卡
7. **评估迭代**：每2周一次代码复现实战 + 30天中期调整

**最终输出**：一份完整的、包含每日任务/方法/工具/评估标准的 `Python数据分析60天学习计划.md`

## 7. 约束与边界

- **不重复**：本 Skill 不详细展开费曼五步法、间隔重复算法、刻意练习细则等——这些由子 Skill 负责
- **只做编排**：本 Skill 的核心价值是"分析需求 + 匹配策略 + 调用子 Skill + 整合为可执行计划"
- **用户可干预**：在每一步都应向用户确认关键决策（尤其是 Step 3 的策略选择），而非完全自动
- **可简化**：如果用户只需要"简单计划"，可跳过部分步骤，直接输出轻量版计划（但仍需保证可执行性）
