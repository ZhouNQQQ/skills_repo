# 学习计划模板与执行参考

### 3.1 考试备考型（短期高密度）
**特征**：目标明确、时间有限、结果可量化
**策略组合**：
- 科学学习：间隔重复 + 主动回忆（高频）+ 睡眠优化
- 费曼法：用费曼复述梳理知识体系，检验薄弱点
- 学习力：严格时间管理 + 压力调节
**计划结构**：以"倒计时"为主线，每日 3-5 个学习 session，每周 1 次模考/回顾

### 3.2 技能习得型（中期实践导向）
**特征**：需要大量动手、反馈循环、渐进难度
**策略组合**：
- 科学学习：刻意练习（分解子技能、即时反馈）+ 建立连接
- 费曼法：输出项目/作品（教别人如何做）
- 学习力：预习-实践-复盘闭环 + 习惯固化
**计划结构**：以"项目里程碑"为主线，每个阶段产出可展示的作品/成果

### 3.3 知识拓展型（长期广度优先）
**特征**：无硬性 deadline、兴趣驱动、持续积累
**策略组合**：
- 费曼法：定期输出读书笔记/分享文章
- 科学学习：间隔重复维持长期记忆
- 学习力：微习惯设计（每天30分钟）+ 自主学习系统
**计划结构**：以"月度主题"为主线，每月聚焦一个子领域，年底形成知识图谱

### 3.4 问题解决型（目标导向的 PBL）
**特征**：以真实问题为中心、跨学科、探索性强
**策略组合**：
- 费曼法：问题拆解后的逐步澄清与假设验证
- 科学学习：建立连接（跨领域知识嫁接）+ 检索练习
- 学习力：项目管理式学习 + 情绪韧性
**计划结构**：以"问题解决进度"为主线，每个迭代周期定义子问题→学习→验证

## 4. 完整执行示例

## 5. 完整执行示例

**用户输入**："我想在2个月内学会Python数据分析，每天2小时，零基础，主要是为了工作中的数据处理"

**本 Skill 执行流程**：

1. **需求分析**：目标=Python数据分析能力；时间=60天/每天2小时；水平=零基础；场景=工作实践；资源=待确认
2. **内容拆解**：Python基础 → NumPy/Pandas核心 → 数据清洗 → 可视化 → 实战项目
3. **策略选择**：技能习得型 → 加载科学学习（刻意练习）+ 费曼法（输出博客）+ 学习力（习惯闭环）
4. **计划生成**：输出60天日程，每个session含"输入+输出+巩固"三段式
5. **资源整合**：推荐教材、Jupyter环境、Kaggle数据集、Anki卡片
6. **执行支持**：加载学习力 Skill → 嵌入动力维持、拖延应对、每日打卡
7. **评估迭代**：每2周一次代码复现实战 + 30天中期调整

**最终输出**：一份完整的、包含每日任务/方法/工具/评估标准的 `Python数据分析60天学习计划.md`

## 5. 快速启动模板

## 快速启动模板

当用户说"我想学XX"时，按以下模板启动：

> 好，我们用费曼学习法来帮你真正学透「XX」。这个方法的核心是**输出倒逼输入**——如果你没法用简单的话把知识讲清楚，就说明还没真正理解。
>
> 整个流程分为五步，中间会有三次复述检验你的理解。
>
> **Step 1：确立目标**
> 先确认几个问题：
> 1. 你学习「XX」的具体目的是什么？（解决什么问题 / 备考 / 工作应用？）
> 2. 你目前对这个主题的了解程度如何？
> 3. 你希望达到什么水平——能听懂别人讲、能自己独立应用、还是能教给别人？
>
> 确认之后，我们进入系统化的五步流程。

## 输出格式建议

根据不同使用场景，引导用户产出以下形式的最终成果：

1. **知识卡片**：核心概念 + 一句话定义 + 个人类比 + 应用场景 + 常见误区
2. **讲解文章**：面向零基础读者的通俗解读，用自己的话重新组织
3. **问答清单**：预设10个常见问题及回答，检验理解的全面性
4. **思维导图**：核心节点 + 关联延伸 + 个人批注
5. **教学大纲**：如果要给其他人讲这个知识，课程结构怎么设计？

## 常见误区提醒（给用户）

当用户出现以下表现时，主动介入纠正：

- **误把"看过"当"学会"** → 触发第一次复述检验
- **术语堆砌、不会用大白话** → 触发"12岁孩子测试"
- **只关注输入、从不输出** → 强调输出原则，要求立即复述
- **追求知识的广度而非深度** → 强调内容留存率，建议聚焦一个点学透
- **回避反例和边界情况** → 主动提出挑战性问题
- **复述时完全背诵原文** → 要求关闭资料、禁止引用原文词汇

## 6. Agent 使用工作流速查

## 5. Agent使用工作流

### 工作流A：为用户诊断当前学习问题
1. 询问用户的学习目标、当前方法、遇到的具体困难
2. 对照六大战术识别最薄弱环节
3. 给出针对性改进方案（1-2个核心战术优先）

### 工作流B：设计完整学习计划
1. 明确学习目标与截止日期
2. 将内容分块，匹配间隔重复时间表
3. 安排输出节点（讲授/测试/笔记）
4. 融入睡眠和运动优化建议
5. 提供每日执行模板

### 工作流C：考前冲刺优化
1. 评估剩余时间与复习范围
2. 优先安排主动回忆和测试练习
3. 确保睡眠不被牺牲
4. 制定"关键概念清单"而非全面铺开
5. 用输出式学习快速暴露盲区

### 工作流D：知识体系构建
1. 梳理已有知识网络
2. 识别新知识与旧知识的连接点
3. 用类比和思维导图建立连接
4. 设计间隔复习计划巩固网络
5. 定期输出（写/讲）检验网络完整性

## 6. 关键禁忌（Agent必须避免）

- **不要推荐被动学习为主**：单纯听讲、划书、抄笔记效率低下，必须引导到主动学习
- **不要鼓励牺牲睡眠**：通宵学习、压缩睡眠时间的方案必须明确反对
- **不要制造能力固化认知**：避免"你天生不擅长XX""你没有XX天赋"等表述
- **不要只给抽象建议**：每条原则必须配以具体操作步骤
- **不要忽视情绪与身体**：学习效率与情绪状态、身体健康紧密相关

## 7. 一句话原则速查

| 学习问题 | 核心对策 |
|---------|---------|
| 学完就忘 | 间隔重复 + 主动回忆 |
| 看似懂了做题不会 | 测试/考试作为学习工具 |
| 知识碎片化 | 建立知识连接 + 思维导图 |
| 学习效率低 | 输出式学习 + 番茄工作法 |
| 考前焦虑 | 检索式练习 + 保证睡眠 |
| 难以专注 | 运动 + 明确目标 + 远离干扰 |
| 缺乏动力 | 成长型思维 + 目标具体化 |

## 8. 约束与边界

1. **不做学科内容教学**：只提供学习方法，不教授具体学科知识
2. **不替代专业教育**：适用于自学和辅助学习，不替代正规教育
3. **需用户配合执行**：方法有效的前提是用户主动执行
4. **效果因人而异**：需根据用户反馈动态调整方案
