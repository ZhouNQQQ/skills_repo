# 连载保障与完本设计

## 七条连载核心原则

1. **节奏可持续**：每章结束必有钩子，维持"期待—满足—新期待"闭环
2. **期待闭环**：短期承诺3章内兑现，中期承诺20-50章兑现，不空许诺
3. **伏笔分层**：按回收周期分短期/中期/长期/身世四层，层层嵌套
4. **信息递进**：每卷揭示的新信息必须颠覆或深化上一卷的认知
5. **冲突储备**：主线外常备2-3条暗线冲突，可随时激活接替
6. **弹性大纲**：卷级大纲固定，章级大纲保留30%调整空间应对读者反馈
7. **收尾规划**：开书时即确定最终卷数与核心谜底，倒推里程碑节点

## 伏笔四层管理

| 层级 | 回收周期 | 功能定位 |
|------|----------|----------|
| 短期伏笔 | 3-5章 | 维持章节追读张力，小悬念高频兑现 |
| 中期伏笔 | 20-50章 | 推动卷级剧情转折，制造阶段性震撼 |
| 长期伏笔 | 百章以上 | 绑定主线核心谜底，全书级回收 |
| 身世暗线 | 贯穿全书 | 主角身份/命运终极揭秘，收尾揭晓 |

**操作要点**：
- 每章至少预埋1个短期钩子，每卷埋设2-3个中期钩子
- 长期伏笔回收前20章开始密集铺垫，避免突兀
- 身世暗线每个阶段释放一个碎片，逐步拼凑全貌
- 所有伏笔必须闭环，未回收的必须在完本前集中处理

## 中期续航三机制

网文中期（30-70%篇幅）最易疲软，激活以下三机制：

1. **地图切换**：当主角实力超越当前区域或资源枯竭时，通过新威胁/新线索/新机遇自然过渡。新地图必须带来全新的规则、势力、资源体系。
2. **威胁升级**：每一卷核心威胁层次递进——从个体危机到群体危机，从物理威胁到存在性威胁。旧威胁解决方式不可直接复用。
3. **势力更迭**：设计"旧势力覆灭—新势力崛起—新势力腐化/新威胁降临"循环。势力重组带来人物关系、阵营立场、权力格局的全面刷新。

**续航公式**：新地图 × 新威胁 × 新势力 = 新鲜感重置

## 完本四大支柱

1. **冲击结尾**：最后一次反转必须颠覆全书最大认知（意义反转，不是情节反转）。主角付出的代价必须真实、不可逆。
2. **伏笔闭环**：所有埋设超过50章的伏笔，在结尾前5章集中回收。采用补叙手法揭示主角隐藏的后手布局。
3. **后手揭晓**：主角在最终冲突中使用的底牌，必须是读者"知道其存在但不知道其用法"的已知元素，禁止凭空新增设定。
4. **情感升华**：核心人物关系在结尾发生质变——和解、决裂、牺牲、传承。情感落点优先于情节落点。

## 卡文急救包

| 场景 | 症状 | 急救方案 |
|------|------|----------|
| 剧情卡死 | 不知道下一步发生什么 | 制造一个突发事件打断当前节奏（刺杀、来信、异变） |
| 节奏拖沓 | 同一冲突写太久，张力下降 | 提前引爆冲突，或插入新变量强制转折 |
| 人设塌缩 | 角色行为不可信 | 回到角色的核心欲望与恐惧，问"此时此刻他最害怕什么" |
| 设定通胀 | 世界观越写越乱，战力崩坏 | 引入新规则限制已有能力，或切换地图重置规则 |
| 动力枯竭 | 读者追更意愿下降 | 激活一个休眠的中期伏笔，制造"原来如此"的阅读快感 |

## 工作流

1. **诊断**：识别用户当前阶段（开篇/中期/后期/完本）和核心痛点
2. **匹配**：从上述模块中提取对应的策略
3. **落地**：提供可立即执行的3-5条具体建议
4. **检查**：如涉及完本设计，使用完本检查清单

## 完本检查清单

- [ ] 所有埋设超过50章的伏笔已回收或明确废弃
- [ ] 主角最终使用的底牌是读者已知的元素
- [ ] 核心人物关系发生质变（和解/决裂/牺牲/传承）
- [ ] 结尾反转颠覆了全书最大认知（意义层面）
- [ ] 主角付出的代价是真实且不可逆的
- [ ] 世界观设定无矛盾，战力体系未崩坏
- [ ] 主要角色的成长弧线已完整闭合
- [ ] 读者的核心疑问得到了回答
