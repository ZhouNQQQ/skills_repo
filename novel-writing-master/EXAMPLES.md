# 工作流示例

## 示例1：从零写末世小说

完整 workflow 覆盖世界观→角色→情节→主题→写作→修订全阶段。

### 阶段一：世界观构建

**1.1 确定末世类型**

| 类型 | 特征 | 经典作品 |
|------|------|----------|
| 核战争 | 辐射、变异、社会崩溃 | 《莱甸路》《疯狂麦克斯》 |
| 病毒瘟疫 | 幸存者挣扎、僵尸化 | 《最后生还者》《行尸走肉》 |
| 气候灾难 | 洪水、干旱、资源枯竭 | 《星际穿越》《启示录》 |
| 外星入侵 | 科技碾压、文明沦陷 | 《世界大战》 |
| AI反叛 | 机器失控、人类边缘化 | 《终结者》《机械姬》 |

**1.2 世界设定清单**

```
☐ 灾难发生时间（前灾/后灾纪年）
☐ 灾难原因与过程
☐ 地理范围（全球/区域/城市）
☐ 当前时间（灾难后多久）
☐ 人口锐减比例
☐ 基础设施状态（电力/网络/交通）
☐ 势力分布（部落/城邦/游民）
☐ 稀缺资源（水/食物/能源/武器/药品）
☐ 社会组织形式
☐ 通讯与信息传递方式
☐ 文化与信仰残留
☐ 语言演变程度
```

→ 详细操作见 [references/02-worldbuilding.md](references/02-worldbuilding.md) + [references/12-apocalyptic.md](references/12-apocalyptic.md)

### 阶段二：角色设计

**2.1 主角弧光**

| 阶段 | 特征 |
|------|------|
| 灾前自我 | 灾前身份、习惯、心理创伤 |
| 末世适应 | 废墟中形成的生存技能与价值观 |
| 觉醒/救赎 | 面对更大危机时的选择与蜕变 |

**2.2 配角矩阵**

```
角色类型     │ 主角关系 │ 功能定位        │ 弧光可能性
─────────────┼─────────┼───────────────┼────────────
导师型       │ 师徒    │ 传授生存技能    │ 可能死亡推动情节
伙伴型       │ 同伴    │ 提供情感支持    │ 可能背叛增加冲突
对手型       │ 敌对    │ 代表不同价值观  │ 可能洗白或对抗到底
牺牲型       │ 深厚联系│ 激发主角潜能    │ 必定死亡制造高潮
新人类型     │ 养育对象│ 代表未来希望    │ 见证成长与传承
```

→ 详细模板见 [references/03-characters.md](references/03-characters.md)

### 阶段三：情节架构

**3.1 三幕结构**

```
第一幕：崩塌世界（25%）
├── 展示末世日常"新常态"
├── 引入主角及生存困境
├── 突发事件打破平衡
└── 主角被迫离开舒适区

第二幕：挣扎与探索（50%）
├── 建立小队/找到临时避难所
├── 遭遇新势力/发现新威胁
├── 主角面临道德困境
├── 多次失败/失去重要之人
└── 最大希望与最大危险的交织

第三幕：抉择与重建（25%）
├── 真相揭示/隐藏的秘密
├── 主角的终极抉择
├── 高潮对决/牺牲与胜利
└── 新秩序的曙光/或永恒的黑暗
```

→ 详细框架见 [references/04-plot.md](references/04-plot.md)

### 阶段四：写作执行

**4.1 写作节奏**

| 阶段 | 每日目标 | 注意事项 |
|------|----------|----------|
| 初稿期 | 1500-3000字 | 先完成，后完美 |
| 修订期 | 1章/天 | 通读后再改 |
| 精修期 | 500-1000字 | 聚焦语言与细节 |

**4.2 场景构建清单**

```
☐ 物理位置与环境描写
☐ 时间（灾前/灾中/灾后多久）
☐ 参与者与他们的状态
☐ 目标：这个场景要实现什么？
☐ 障碍：什么阻止目标达成？
☐ 冲突类型：外部/内部/人际
☐ 结果：目标达成/未达成/代价
☐ 遗留问题：连接下一个场景
```

### 阶段五：验证与回溯（会话结束前）

```
☐ 设定一致性：新内容与已有知识库无冲突
☐ 剧情bug：时间线、人物行为逻辑、战力体系一致
☐ 伏笔状态：本次回收的伏笔已标记，新增的已记录预期回收位置
☐ 变更摘要：已列出所有知识库更新项
☐ 角色一致性：新对话/行为符合既定人设
```

→ 验证标准见 [references/13-knowledge-base.md](references/13-knowledge-base.md)

## 示例2：卡文急救

**场景**：写到第15章突然不知道接下来怎么发展

**Workflow**：

1. **读取知识库**：打开 `foreshadowing-log.md`，查看所有未回收的伏笔
2. **寻找素材**：在伏笔表中找3-5个近期可回收的伏笔
3. **调用连载模块**：运行 [references/11-serialization.md](references/11-serialization.md) 的卡文急救流程
4. **节奏诊断**：对照 [references/14a-plot-pressure-model.md](references/14a-plot-pressure-model.md) 检查当前章型是否需要切换
5. **生成方案**：基于可回收伏笔+章型切换需求，生成2-3个可行的下章走向
6. **用户选择**：展示选项，用户选定后继续创作
7. **同步知识库**：将新确定的走向记录到 plot.md 和 foreshadowing-log.md

**关键原则**：卡文时**不要硬写**，先读知识库找素材，90%的卡文是因为忘记了之前埋下的伏笔。

## 示例3：多章 Consistency 检查

**场景**：写完第20-30章后，怀疑前面有设定冲突

**Workflow**：

1. **读取完整知识库**：novel-bible.md + worldbuilding.md + characters.md + consistency-log.md
2. **逐项比对**：
   - 角色档案：每个出场角色的行为是否符合人设？战力是否有异常波动？
   - 世界观设定：地理、时间线、势力关系是否有矛盾？
   - 伏笔追踪：已埋伏笔是否被意外遗忘或错误回收？
3. **标记问题**：在 consistency-log.md 中记录发现的冲突
4. **生成修复方案**：按优先级排序（P1: 剧情bug > P2: 设定不一致 > P3: 细节偏差）
5. **修复并同步**：修改章节正文 + 更新知识库 + 记录修复决策
6. **验证**：重新读取修复后的内容，确认无新增冲突

**输出格式**：表格清单，包含问题位置、严重级别、修复建议、修复状态。
